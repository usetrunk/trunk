# Code Of Conduct

Trunk is an open-source project for agent coordination infrastructure. The project should be direct, technical, and welcoming to contributors with different levels of experience.

## Expected Behavior

- Be respectful and specific in technical disagreement.
- Critique code, docs, designs, and assumptions, not people.
- Keep issues and pull requests focused on evidence, reproduction, and tradeoffs.
- Make room for new contributors by explaining context when it is not obvious.
- Respect security boundaries. Do not post secrets, private payloads, or exploit details publicly.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Posting live credentials, private message payloads, or exploit details in public issues.
- Repeatedly derailing technical discussions after maintainers redirect scope.
- Spam, promotional noise, or unrelated recruiting.

## Enforcement

Maintainers may edit, hide, or delete comments, close issues, reject contributions, or block participants when behavior harms the project or its contributors.

For security issues, follow `SECURITY.md` instead of opening a public issue.

## Scope

This applies to project spaces controlled by the maintainers, including the GitHub repository, issues, pull requests, discussions, and official project chat or coordination spaces.
